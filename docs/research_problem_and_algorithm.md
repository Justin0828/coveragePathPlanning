# 面向带摆臂圆盘作业机器人的覆盖路径规划：研究问题、相关工作与算法实现

> 本文档依据当前仓库代码撰写，目标是作为论文中“问题定义、相关工作、方法设计与实现细节”章节的技术底稿。文中严格区分：经典覆盖路径规划思想、本项目采用的工程启发式，以及当前实现尚不能声称的理论保证。

## 1. 一句话概括这个项目

本项目研究的问题是：给定一张包含障碍物的二维二值地图，以及一台“车体 + 可转动摆臂 + 圆盘形作业端”的机器人，怎样把可作业区域拆成若干容易覆盖的轴对齐矩形，决定这些矩形的访问次序，再在每个矩形内生成往复式轨迹，最终尽可能覆盖全部自由空间，并输出可复现实验结果。

直观上，它像让一台带长柄圆盘的割草车完成一块有障碍物的场地：

1. 先把不规则场地切成若干块长方形“责任田”；
2. 把能相互到达的责任田连成图；
3. 决定先割哪块、后割哪块，必要时借道已经割过的块；
4. 在每块田里按“去一趟、错开一条带、反向回来”的牛耕方式作业；
5. 将机器人运动离散成位姿序列，用圆盘扫过的像素计算真实覆盖率。

这不是单个算法，而是一条分层规划流水线：

```text
二值地图
  -> 保守栅格化
  -> 行/列扫描矩形候选
  -> 带数量与重叠惩罚的贪婪选择
  -> 随机扩展补洞
  -> 矩形邻接连通性修复
  -> 邻接图上的近似全局访问顺序
  -> 矩形进/出点设计
  -> 矩形内往复式覆盖
  -> 转弯、直行和摆臂动作离散化
  -> 覆盖掩膜、覆盖率、图片、视频和 JSON 指标
```

## 2. 研究问题

### 2.1 直觉描述

普通的点到点路径规划只要求机器人从起点安全到达终点。覆盖路径规划（Coverage Path Planning, CPP）要求作业工具扫过目标区域中的每一个位置，因此不仅关心“能不能到”，还关心：

- 是否遗漏自由区域；
- 是否反复经过同一区域；
- 总行驶距离、转弯次数和作业时间是否足够小；
- 轨迹是否符合机器人车体、摆臂和作业圆盘的几何及速度约束；
- 规划结果能否在不同地图、参数和随机种子下复现。

如果直接在整张地图上联合优化连续轨迹、区域覆盖、避障和机械臂动作，问题规模很大。当前项目采用“先分区、再排序、后做局部覆盖”的分层思路，将一个复杂连续优化问题拆成三个较容易处理的子问题：矩形覆盖选择、图上的访问顺序、矩形内轨迹生成。

### 2.2 环境模型

设输入灰度图像为

$$
M\in\{0,255\}^{H_p\times W_p},
$$

其中 $M_{uv}=0$ 表示障碍物，$M_{uv}=255$ 表示自由空间；每个像素对应的世界长度为 $s_p>0$ 米。图像读入后上下翻转，使数组第 0 行对应世界坐标系的底部。于是世界坐标与像素坐标近似满足

$$
(u,v)=\left(\left\lfloor\frac{x}{s_p}\right\rfloor,
              \left\lfloor\frac{y}{s_p}\right\rfloor\right).
$$

定义像素自由集与连续自由区域为

$$
\mathcal P_F=\{(u,v)\mid M_{vu}=255\},\qquad
\mathcal F\subset\mathbb R^2.
$$

算法首先用边长为 $g$ 个像素的粗栅格重新离散地图。每个粗栅格的物理边长为

$$
s_g=g s_p.
$$

只要一个粗栅格包含至少一个黑色像素，代码就将整个栅格标记为障碍物。这是一种保守栅格化：牺牲部分可用面积，换取矩形候选不穿过已知障碍物。

### 2.3 机器人模型

机器人状态可写成

$$
q(t)=\bigl(x_c(t),y_c(t),\theta(t),\phi(t)\bigr),
$$

其中 $(x_c,y_c)$ 是车体中心，$\theta$ 是车体航向角，$\phi$ 是摆臂相对角。机器人参数包括：

- 圆盘半径 $r_d$；
- 摆臂长度 $L$；
- 车宽 $w_c$；
- 车身半长 $h_c$；
- 车中心到摆臂枢轴的后向距离 $d_p$；
- 最大线速度 $v_{\max}$；
- 最大车体角速度 $\omega_{\max}$；
- 摆臂角度范围 $[\phi_{\min},\phi_{\max}]$；
- 最大摆臂角速度 $\omega_{a,\max}$。

令 $\mathbf e(\gamma)=(\cos\gamma,\sin\gamma)$，则枢轴位置和圆盘中心分别为

$$
\mathbf p_{\text{pivot}}(q)
=\mathbf p_c-d_p\mathbf e(\theta),
$$

$$
\mathbf p_d(q)
=\mathbf p_c-d_p\mathbf e(\theta)
 +L\mathbf e(\theta+\pi+\phi).
$$

某一位姿下的瞬时作业区域是以圆盘中心为圆心、$r_d$ 为半径的闭圆盘：

$$
\mathcal D(q)=\{\mathbf z\in\mathbb R^2:
\|\mathbf z-\mathbf p_d(q)\|_2\le r_d\}.
$$

轨迹 $q:[0,T]\to\mathcal Q$ 的连续扫掠区域为

$$
\mathcal C(q)=\bigcup_{t\in[0,T]}\mathcal D(q(t)).
$$

理想覆盖约束是 $\mathcal F\subseteq\mathcal C(q)$。但当前实现不是求解这个连续约束下的全局最优轨迹，而是最大化离散估计覆盖率，同时用启发式降低矩形数量、重叠和矩形之间的转场代价。

### 2.4 输出与评价指标

算法输出：矩形集合 $\mathcal R$、邻接图 $G$、矩形访问序列 $\pi$、离散位姿序列 $Q=(q_0,\ldots,q_{N-1})$，以及覆盖结果图和视频。

离散覆盖像素集为

$$
\widehat{\mathcal P}_C
=\bigcup_{k=0}^{N-1}
\left\{p:\|p-\Pi(\mathbf p_d(q_k))\|_2\le r_{d,p}\right\},
$$

其中 $\Pi$ 是世界坐标到像素坐标的映射，代码使用

$$
r_{d,p}=\max\left(1,\left\lfloor\frac{r_d}{s_p}\right\rfloor\right).
$$

最终覆盖率为

$$
\rho
=\frac{|\widehat{\mathcal P}_C\cap\mathcal P_F|}
       {|\mathcal P_F|}.
$$

项目同时记录自由像素数、已覆盖自由像素数、未覆盖自由像素数、矩形数、邻接边数、重访次数、位姿数、估计时长和各阶段运行时间。

### 2.5 可以写进论文的联合目标，以及代码实际如何近似它

从研究问题看，一个自然的多目标形式是

$$
\max_q\quad
\rho(q)-\lambda_T T(q)-\lambda_R N_R(q)-\lambda_O A_{\text{overlap}}(q),
$$

并满足自由空间、碰撞、速度和关节范围约束。当前代码没有联合求解该式，而是依次使用三个代理目标：

1. 分区阶段奖励新覆盖面积，惩罚矩形数量和矩形重叠；
2. 全局排序阶段最小化矩形覆盖时间代理与矩形间转移时间代理；
3. 局部阶段按照机器人几何生成固定间距的往复式轨迹。

因此论文中应称其为“分层启发式求解框架”，不应称为联合全局最优方法。

## 3. 相关工作与本项目的位置

### 3.1 覆盖路径规划

Choset 将覆盖规划概括为让机器人的传感器或执行器经过目标自由空间，并系统梳理了精确细胞分解、近似细胞分解和基于图的覆盖方法 [1]。Galceran 与 Carreras 后来从离线/在线方法、环境表示、单机/多机和优化指标等角度给出了更完整的综述 [2]。本项目属于已知二维地图上的离线、单机器人、近似分解式覆盖规划。

### 3.2 Boustrophedon 分解与往复式覆盖

Choset 与 Pignon 提出的 Boustrophedon Cellular Decomposition 在扫描线与障碍物相交拓扑发生变化的临界位置切分环境，使每个细胞内部可以用平行往复线覆盖 [3]。Acar 等进一步用 Morse 函数统一描述覆盖分解 [4]；Huang 研究了扫描方向对分解和代价的影响 [5]。

本项目借用了“细胞内牛耕式往复覆盖”的核心直觉，但必须注意：项目的分区阶段不是经典 Boustrophedon Cellular Decomposition。经典方法根据扫描线连通分量的拓扑事件产生不重叠细胞；本项目从栅格地图生成大量轴对齐矩形候选，再用带重叠惩罚的贪婪算法选择矩形，所得矩形允许重叠，随后还有随机补洞和连通修复。论文中更准确的名称是：

> 基于栅格矩形候选选择的近似区域分解，以及矩形内 Boustrophedon-style 覆盖。

### 3.3 集合覆盖、最大覆盖与贪婪选择

把每个候选矩形视为一个像素集合后，矩形选择与集合覆盖/最大覆盖问题直接相关。Chvátal 分析了集合覆盖的经典贪婪启发式 [6]；Nemhauser、Wolsey 与 Fisher 研究了子模集合函数的贪婪近似 [7]。本项目的目标包含负的矩形数量项与重复覆盖项，因而一般不是标准的单调最大覆盖问题，代码采用“选择当前正边际收益最大的候选，直到不存在正收益”的启发式，不能直接继承经典集合覆盖或单调子模最大化的近似比。

### 3.4 基于图和生成树的覆盖

生成树常被用于保证连通区域的系统性遍历。Gabriely 与 Rimon 的 Spanning Tree Coverage 在栅格化自由空间上构建生成树并沿树组织连续覆盖 [8]。本项目也使用最小生成树，但树的节点是已选择的矩形而不是自由栅格；树只负责给出矩形访问顺序，矩形内部仍由独立的往复式轨迹覆盖。因此它与 STC 相关，但不是 STC 的直接实现。

全局顺序还与旅行商问题及图上的覆盖游走相关。Prim 算法提供最小生成树 [9]，基于生成树的加倍/深度优先访问是旅行商启发式的经典来源 [10]。本项目构造一条开放式树遍历：除从起点到最终节点的分支外，其余树边允许往返，以保证连通分量中的每个矩形至少被访问一次。

### 3.5 本项目的技术组合

相对于上述工作，当前实现的特点不在于提出了全新的单一理论算法，而在于将下列组件组合成适配特殊机器人几何的可复现实验系统：

- 行、列两个方向的矩形候选生成；
- 同时惩罚矩形数量和重复覆盖的贪婪选择；
- 未覆盖栅格上的随机最大矩形扩展；
- 用候选矩形桥接分离的邻接分量；
- “随机化近邻 Hamilton 路径候选 + 最小生成树保底游走”的双候选全局排序；
- 针对车体—枢轴—摆臂—圆盘几何推导的作业带间距；
- 矩形间进/出点协调、重复节点借道和离散位姿生成；
- 由同一个圆盘覆盖掩膜产生指标、最终图片和视频。

## 4. 总体算法

设最终矩形数为 $n$。流水线的核心过程可写成：

```text
Algorithm: Hierarchical Coverage Planning
Input : binary map M, map resolution sp, grid size g,
        segmentation parameters (lmin, alpha, beta), robot parameters, sample rate f
Output: rectangles R, graph G, visit order pi, pose sequence Q, coverage ratio rho

1  B <- rasterize(M, g)
2  Crow <- generate row-scan rectangle candidates from B
3  Srow <- positive-marginal greedy selection(Crow)
4  Bmasked <- mark Srow as unavailable in B
5  Ccol <- generate column-scan candidates from Bmasked
6  Scol <- positive-marginal greedy selection(Ccol | initial coverage=Srow)
7  S <- Srow union Scol
8  S <- random expansion on uncovered free cells
9  S <- connectivity repair(S, Crow union unmasked-column-candidates)
10 G <- rectangle adjacency graph of S
11 s <- configured start, or largest rectangle
12 pi_MST <- open DFS walk on a transition-weighted MST of G
13 pi_greedy <- best complete randomized unvisited-neighbor path found in 449 trials
14 pi <- lower-cost valid candidate among pi_MST and pi_greedy
15 assign an entry and exit boundary point to every occurrence in pi
16 for each rectangle occurrence in pi:
17     move robot to its entry pose
18     if first visit: generate boustrophedon-style coverage poses
19     else          : only transit through the revisited rectangle
20 C <- union of discretized disc footprints over all poses
21 rho <- |C intersect free_pixels| / |free_pixels|
```

## 5. 阶段一：地图栅格化与矩形分解

### 5.1 保守栅格化

粗栅格 $B\in\{0,1\}^{H\times W}$ 中，$1$ 表示障碍。对每个 $g\times g$ 像素块 $P_{ij}$，预期规则为

$$
B_{ij}=\mathbb I\left[\exists p\in P_{ij},\ M(p)=0\right].
$$

这确保粗栅格矩形只由完全不含黑色障碍像素的块组成。若原图尺寸不能被 $g$ 整除，代码会扩展数组到 $g$ 的整数倍。当前实现已验证将扩展区误当成自由空间，可产生超出原图范围的矩形，见第 14 节第 3 项。

### 5.2 行扫描候选

对每个扫描位置，算法估计从当前自由单元沿一个轴可以延伸的高度，再向垂直轴两侧扩展，得到一个轴对齐自由矩形。抽象地，一个候选为

$$
R_i=[x_i^1,x_i^2]\times[y_i^1,y_i^2],
$$

并满足

$$
x_i^2-x_i^1\ge \ell_{\min},\qquad
y_i^2-y_i^1\ge \ell_{\min},\qquad
R_i\cap\mathcal O_g=\varnothing,
$$

其中 $\ell_{\min}$ 是最小边长，$\mathcal O_g$ 是粗栅格障碍区域。

代码对每行、每列位置都尝试生成候选。它维护当前位置对应的自由高度 $h$，再寻找左右相邻位置中高度不少于 $h$ 的最大连续区间。该矩形随后从栅格坐标转换为世界坐标。对每条扫描线，算法先按坐标去除重复矩形，再按面积降序排列，面积相同时按坐标字典序稳定打破平局，最后保留前 `candidate_top_k` 个送入贪婪选择。这是每条扫描线的上限，不是整张地图的全局上限。为避免截断恰好删除必要的桥接矩形，邻接连通修复仍使用未经 top-k 截断的完整去重候选池。

### 5.3 第一次贪婪选择

令候选全集为 $\mathcal C$，已选集合为 $S\subseteq\mathcal C$。对每个像素 $p$，定义覆盖计数

$$
k_p(S)=\sum_{R\in S}\mathbb I[p\in R].
$$

项目实际对应的矩形选择目标可以写成

$$
J_{\text{seg}}(S)
=s_p^2\sum_p\mathbb I[k_p(S)\ge1]
-\alpha|S|
-\beta s_p^2\sum_p\max(k_p(S)-1,0).
$$

三项分别表示：

1. 至少被一个矩形覆盖的面积；
2. 每增加一个矩形的固定惩罚 $\alpha$；
3. 每个像素超过第一次后的重复覆盖惩罚，权重为 $\beta$。

给定新候选 $R$，令

$$
A_{\text{new}}(R\mid S)
=s_p^2|\{p\in R:k_p(S)=0\}|,
$$

$$
A_{\text{overlap}}(R\mid S)
=s_p^2|\{p\in R:k_p(S)\ge1\}|.
$$

则代码计算的边际收益恰好是

$$
\Delta J(R\mid S)
=A_{\text{new}}(R\mid S)-\alpha
-\beta A_{\text{overlap}}(R\mid S).
$$

每轮选择边际收益最大的候选：

$$
R^*=\arg\max_{R\in\mathcal C\setminus S}\Delta J(R\mid S).
$$

若 $\Delta J(R^*\mid S)\le0$ 则停止，否则令 $S\leftarrow S\cup\{R^*\}$ 并更新覆盖计数。这解释了两个参数的物理意义：$\alpha$ 的单位应与面积一致，用于控制矩形数量；$\beta$ 是重复面积的无量纲权重。

### 5.4 列扫描补充

只做一个扫描方向容易遗漏狭长或方向相反的区域。项目将第一阶段选中的矩形在粗栅格中暂时标成障碍，再从列方向生成候选。第二次贪婪选择以第一阶段的覆盖计数作为初值，因此新矩形只有在补充的新面积足以抵消固定惩罚和重叠惩罚时才会入选。

这相当于一种方向互补策略：先用行扫描解释主要自由空间，再让列扫描只关注剩余空洞。

### 5.5 随机扩展补洞

贪婪选择结束后，算法反复计算尚未被矩形覆盖的自由粗栅格集合 $U$。若 $U\ne\varnothing$，使用固定随机种子均匀抽取一个单元 $(i,j)\in U$，然后：

1. 在该行向左右扩展到障碍或边界；
2. 固定所得水平区间，向上下扩展，只要整条区间保持自由；
3. 将得到的矩形加入集合，前提是两条边均不小于 $\ell_{\min}$。

成功加入矩形后连续失败计数清零；若生成的矩形太小则失败计数加一；连续失败 5 次即停止。扩展使用原始粗栅格而不是已选矩形掩膜，所以新矩形允许与已有矩形重叠，但其采样种子点必定位于此前未覆盖区域。这一步不是概率完备算法，也不保证消除所有空洞。

### 5.6 邻接连通性修复

贪婪与随机补洞只关心面积，不保证矩形邻接图连通。设当前矩形集合在邻接关系下分成连通分量

$$
\mathcal K=\{K_1,\ldots,K_m\}.
$$

算法在未选择的行扫描候选和“未掩膜地图上的列扫描候选”中寻找桥接矩形 $R$。若 $R$ 同时与至少两个不同分量相邻，则它是可行桥。代码选择面积最小的可行桥：

$$
R^*=\arg\min_R \operatorname{area}(R),
$$

将其加入集合后重新计算连通分量，直到图连通或不存在单个候选可以同时连接两个分量。因而该步骤是局部贪婪修复，不保证使用最少桥、不保证桥面积总和最小，也不保证最终一定连通。

## 6. 阶段二：矩形邻接图与全局访问顺序

### 6.1 邻接图

对最终矩形集合 $\mathcal R=\{R_0,\ldots,R_{n-1}\}$，构造无向图

$$
G=(V,E),\qquad V=\{0,\ldots,n-1\}.
$$

若两个矩形相交或接触部分足够大，则在它们之间连边。对正常的相交/接触情形，令

$$
\delta_x=\min(x_i^2,x_j^2)-\max(x_i^1,x_j^1)\ge0,
$$

$$
\delta_y=\min(y_i^2,y_j^2)-\max(y_i^1,y_j^1)\ge0.
$$

当前代码使用交叠“矩形”的对角线作为阈值：

$$
(i,j)\in E
\iff \sqrt{\delta_x^2+\delta_y^2}>\ell_{\min}.
$$

因此：共享竖直边时判据退化为共享边长度 $\delta_y>\ell_{\min}$；共享水平边时为 $\delta_x>\ell_{\min}$；有面积重叠时则使用交叠区域对角线。代码还包含一个针对严格分离矩形的额外分支，但它的几何判据并不等价于标准“共享边”定义，论文方法描述宜采用上式，并在实现局限中披露该分支。

邻接矩阵 $A\in\{0,1\}^{n\times n}$ 满足

$$
A_{ij}=A_{ji}=\mathbb I[(i,j)\in E],\qquad A_{ii}=0.
$$

### 6.2 起始矩形

若配置显式给出起始编号 $s$，代码检查 $0\le s<n$ 后直接使用；否则选择面积最大的矩形：

$$
s=\arg\max_i (x_i^2-x_i^1)(y_i^2-y_i^1).
$$

全局排序只处理从 $s$ 可达的连通分量 $V_s$。如果连通修复仍失败，其他分量不会进入访问序列，也就不会被局部覆盖。

### 6.3 矩形覆盖时间代理

设摆臂可用的对称最大摆角为

$$
\psi=\min(|\phi_{\min}|,|\phi_{\max}|,90^\circ),
$$

则单条可变摆臂作业带的最大横向覆盖宽度估计为

$$
W_{\max}=2(L\sin\psi+r_d).
$$

对宽 $w_i$、高 $h_i$ 的矩形，短边决定轨迹条带数，长边决定单条作业距离：

$$
(n_i^{\text{lane}},\ell_i)=
\begin{cases}
(\max(1,\lceil h_i/W_{\max}\rceil),w_i),&w_i\ge h_i,\\
(\max(1,\lceil w_i/W_{\max}\rceil),h_i),&w_i<h_i.
\end{cases}
$$

覆盖时间代理为

$$
c_i^{\text{cov}}
=\frac{n_i^{\text{lane}}\ell_i}{\bar v}
+\frac{(n_i^{\text{lane}}-1)180^\circ}{\bar\omega}.
$$

这里假设每增加一条作业带需要一次 $180^\circ$ 重定向。$\bar v$、$\bar\omega$ 和 $b$ 所需的几何量均从当前实验的机器人配置读取，图排序与后续位姿生成使用同一组机器人参数。但该代价仍只是排序代理，不是由最终位姿序列严格计算的真实时间。

### 6.4 矩形转移时间代理

令矩形中心为

$$
\mathbf c_i=\left(\frac{x_i^1+x_i^2}{2},
                       \frac{y_i^1+y_i^2}{2}\right).
$$

相邻矩形间的转移代价为

$$
c_{ij}^{\text{tr}}
=\frac{\|\mathbf c_i-\mathbf c_j\|_2}{\bar v}
+\frac{180^\circ}{\bar\omega}.
$$

它用中心距离近似真实入口到入口距离，并固定增加一次 $180^\circ$ 转向时间。该权重对称，因此图是无向加权图。

### 6.5 访问序列的目标

访问序列允许重复节点：

$$
\pi=(\pi_0,\ldots,\pi_m),\qquad
\pi_0=s,\quad (\pi_k,\pi_{k+1})\in E.
$$

所有可达矩形至少出现一次：

$$
V_s\subseteq\{\pi_0,\ldots,\pi_m\}.
$$

代码用于比较候选路线的总代价为

$$
J_{\text{route}}(\pi)
=\sum_{i\in\operatorname{unique}(\pi)}c_i^{\text{cov}}
+\sum_{k=0}^{m-1}c_{\pi_k\pi_{k+1}}^{\text{tr}}.
$$

因为每条完整候选路线覆盖相同的可达节点集合，第一项在候选之间是常数；实际选择主要由转场代价之和决定。允许重复节点使该问题更接近“从固定起点出发、终点自由的最短图覆盖游走”，而不是要求每个节点恰好访问一次的标准 TSP。

### 6.6 最小生成树保底路线

为了保证任意连通邻接图都能得到完整路线，代码先用 $c_{ij}^{\text{tr}}$ 作为边权，通过 Prim 算法构造以 $s$ 为根的最小生成树 $T$。

若直接深度优先遍历并返回起点，每条树边恰好走两次。当前实现允许终点自由，因此在树中寻找距 $s$ 加权距离最大的节点

$$
t^*=\arg\max_{v\in V_s}d_T(s,v).
$$

对不在唯一路径 $P_T(s,t^*)$ 上的分支执行“进入—遍历—返回”，而将 $P_T(s,t^*)$ 放在最后且不折返。所得开放式游走的树边代价为

$$
C_{\text{tree-walk}}
=2\sum_{e\in E_T}w(e)-d_T(s,t^*).
$$

它一定访问树中全部节点；每条非最终分支边经过两次，最终分支边经过一次。这里的“最远终点”只是在已经选定的 MST 上使该开放式 DFS 游走最短，不代表它是原图全部覆盖游走的全局最优解。

### 6.7 随机化无重访候选

MST 路线可靠但可能产生很多重访。代码另外搜索不重复节点的完整路径。给定当前节点 $i$ 和已访问集合 $S_v$，候选下一节点必须满足

$$
j\notin S_v,\qquad A_{ij}=1.
$$

对每个候选计算二元排序键

$$
\left(c_{ij}^{\text{tr}}, d_U(j;S_v)\right),
$$

其中

$$
d_U(j;S_v)=\sum_k A_{jk}\mathbb I[k\in V_s\setminus S_v]
$$

是 $j$ 尚未访问的邻居数。先偏好转场便宜的节点；代价相同时，偏好剩余出口少的节点，以降低后续陷入死路的概率，这与 Warnsdorff 类启发式的直觉相似。

算法并非始终选择第一名，而是在排序后的前 $K$ 名中随机抽一个。当前固定执行：

- $K=1$：1 次纯贪婪；
- $K=2$：128 次；
- $K=3$：160 次；
- $K=4$：160 次。

合计 449 次，且每次使用由常数 42、$K$ 和试验编号确定的局部随机种子，所以结果可复现。只有访问全部 $V_s$ 的候选才被接受。最后将全部完整无重访候选与 MST 保底路线按 $J_{\text{route}}$ 比较，返回代价最低者。若图不存在从 $s$ 开始的 Hamilton 路径，随机化分支可以全部失败，但 MST 路线仍然有效。

## 7. 阶段三：矩形进出点与矩形间转场

### 7.1 相邻矩形的边界过渡点

对于沿竖直边相接的矩形 $R_i,R_j$，共享边上的可用纵坐标区间为

$$
I_y=[\max(y_i^1,y_j^1),\min(y_i^2,y_j^2)].
$$

若没有参考点，选择中点 $y=(\inf I_y+\sup I_y)/2$；若当前矩形已经有入口参考点 $p^{\text{in}}$，则将它的纵坐标投影到共享区间：

$$
y=\operatorname{clip}(p_y^{\text{in}},\inf I_y,\sup I_y).
$$

水平共享边同理。这样尽量让当前矩形的入口和出口在同一横向/纵向位置，减少矩形内部为转场产生的侧向移动。

如果两个“相邻”矩形没有精确共享边，例如它们有面积重叠，代码退化为：分别从两个矩形中心沿对方中心方向取各自边界点。这两个点不一定重合。

### 7.2 首、中、末矩形的规则

对访问序列中的每一次出现分别构造 $(p_k^{\text{in}},p_k^{\text{out}})$：

- 只有一个矩形时，沿矩形长轴选择一对相对边中点；
- 第一个矩形的出口朝向下一个矩形，入口取朝向下一个矩形的反方向边界点；
- 中间矩形的入口来自前一对矩形的过渡点，出口来自后一对矩形的过渡点，并用入口坐标进行投影；
- 最后一个矩形的入口来自前一矩形，出口取背离前一矩形的边界点；
- 若内部矩形算出的入口与出口重合，则将出口改为背离前一矩形的边界点。

### 7.3 边界点对应的机器人位姿

当圆盘中心位于矩形边界点 $p$ 且摆臂角为 $0^\circ$ 时，车体沿边界外法向停靠。令

$$
D=d_p+L.
$$

左、上、右、下边界分别对应航向 $180^\circ,90^\circ,0^\circ,270^\circ$；车中心距圆盘中心 $D$。例如右边界点 $(x_i^2,y)$ 对应

$$
\theta=0^\circ,\qquad
\mathbf p_c=(x_i^2+D,y),\qquad \phi=0^\circ.
$$

若边界点恰好在角点，代码按照左、上、右、下的判断顺序选择第一种匹配姿态。

### 7.4 矩形间运动

从当前车中心 $a$ 到目标入口车中心 $b$，代码执行：

1. 原地转到 $\operatorname{atan2}(b_y-a_y,b_x-a_x)$；
2. 以最大线速度沿直线移动到 $b$；
3. 原地转到入口所要求的四向航向；
4. 将摆臂设置为入口姿态。

该转场是纯几何插值，没有对车体、摆臂或圆盘做障碍物碰撞检测，也没有强制直线位于相邻矩形并集内。因此“矩形图相邻”目前不等价于“生成的完整机器人转场一定无碰撞”。

访问序列若重复经过某个矩形，第一次访问时执行覆盖；之后的出现只作为图上的借道节点，从本次入口运动到本次出口，不再次执行矩形内覆盖。

## 8. 阶段四：矩形内往复式覆盖

### 8.1 作业带宽的几何来源

当前局部覆盖使用可变摆臂作业带，而不是固定的 `half_band_width`。令

$$
\psi=\min(|\phi_{\min}|,|\phi_{\max}|,90^\circ),
$$

则一条主作业线在横向上最多可覆盖

$$
W_{\max}=2(L\sin\psi+r_d).
$$

若矩形短边为 $s$，条带数取

$$
n=\max(1,\lceil s/W_{\max}\rceil),
$$

每条带宽为 $s/n$。对单条宽度 $w_b$，代码选择足以覆盖该条带的最小对称摆幅

$$
\alpha=\arcsin\left(\frac{w_b/2-r_d}{L}\right),
$$

并将其裁剪到 $[0,\psi]$。当 $w_b\le2r_d$ 时，$\alpha=0^\circ$，即圆盘直径本身足以覆盖该条带；当区域足够宽时，摆臂可接近或达到 $[-90^\circ,90^\circ]$ 的最大摆幅。默认参数 $L=1.2\ \mathrm m,r_d=0.75\ \mathrm m,\psi=90^\circ$ 给出 $W_{\max}=3.9\ \mathrm m$。

### 8.2 起始角点

算法根据入口点相对于矩形中心的左右、上下位置选择最近象限中的角点，编号为：左上 0、左下 1、右上 2、右下 3。当前实现把圆盘中心放在该角点，并让摆臂达到允许的最大对称摆角。例如左上角使用 $\theta=0^\circ,\phi=-\psi$，车中心由圆盘中心、航向、枢轴偏置和摆臂长度反解得到。

### 8.3 从入口到起始点

边界入口姿态与角点起始姿态之间用一条轴对齐 L 形路径连接：入口车中心 $n_0$、一个中间直角点 $n_1$、起始车中心 $n_2$。在每条线段前先原地转向，然后直行；到达起始车中心后转到 $0^\circ$ 或 $180^\circ$ 的主扫描方向，并把摆臂转到角点所需的 $\pm\psi$。

### 8.4 牛耕节点序列

当前端到端流水线生成水平主作业线。矩形高度 $h$ 先按第 8.1 节的 $W_{\max}$ 分成 $n$ 条带，每条带宽 $w_b=h/n$。第 $k$ 条带的中心线位于该条带上下边界的中点，摆臂从 $-\alpha_k$ 摆到 $+\alpha_k$ 或反向摆动，其中 $\alpha_k$ 由该条带宽度和圆盘半径决定。相邻条带的行驶方向交替，形成牛耕式来回覆盖。

每条主作业线的圆盘中心横向端点向矩形内部偏置不超过 $r_d$，以减少圆盘中心越界；完成最后一条带后，再转场到用户指定的出口边界姿态。

### 8.5 摆臂动作

对每条主作业线，算法先把车体和摆臂调整到该条带的起始姿态。直线行驶过程中，摆臂不再只从一侧线性摆到另一侧一次，而是以固定幅值 $\omega_{a,\max}$ 在 $[-\alpha_k,+\alpha_k]$ 之间形成反射三角波，直到 pivot 到达该作业线终点。

一次半摆的时间和作业安全车速为

$$
T_{\mathrm{half}}=\frac{2\alpha_k}{\omega_{a,\max}},
\qquad
v_{\mathrm{work}}=
\min\left(v_{\max},\frac{2r_d}{T_{\mathrm{half}}}\right).
$$

因此相邻半摆之间 pivot 的纵向前进距离不超过圆盘直径，避免往复中心曲线之间出现大于圆盘直径的空隙。窄条带若 $\alpha_k=0$，则摆臂保持不动并使用正常速度上限。

条带之间的转场仍按“原地转车体、直线移动、必要时原地转摆臂”的分段模型生成。位姿序列中的 `arm_angle` 会随时间变化，覆盖掩膜、最终图片和视频绘制均使用该角度计算圆盘中心。

## 9. 连续动作的离散位姿实现

### 9.1 航向角

从点 $a$ 指向点 $b$ 的航向为

$$
\theta_{ab}=\operatorname{deg}(\operatorname{atan2}(b_y-a_y,b_x-a_x))\bmod360^\circ.
$$

原地转弯总是采用最短有符号角：

$$
\Delta\theta=((\theta_1-\theta_0+180^\circ)\bmod360^\circ)-180^\circ.
$$

### 9.2 采样规则

采样频率为 $f$。长度为 $d$ 的直线段所需时间和样本数为

$$
T_m=\frac d{v_{\max}},\qquad
N_m=\max(1,\lceil fT_m\rceil).
$$

第 $k$ 个样本位置为

$$
\mathbf p_k=\mathbf p_0+\frac{k}{N_m}(\mathbf p_1-\mathbf p_0),
\qquad k=1,\ldots,N_m.
$$

角度为 $|\Delta\theta|$ 的转弯使用

$$
T_r=\frac{|\Delta\theta|}{\omega_{\max}},\qquad
N_r=\max(1,\lceil fT_r\rceil),
$$

并线性插值航向。摆臂转动同理：

$$
T_a=\frac{|\phi_1-\phi_0|}{\omega_{a,\max}},\qquad
N_a=\max(1,\lceil fT_a\rceil).
$$

每个位姿字典保存航向、摆臂角、车速、车体角速度和车中心位置。位姿没有显式时间戳，默认相邻样本间隔 $1/f$。

### 9.3 初始状态处理

虽然配置包含机器人初始位置和初始航向，但全局位姿生成对第一个矩形直接将工作机器人复制为该矩形入口姿态，并添加一个静止样本；没有从配置初始位置规划到第一个入口。这意味着当前实验轨迹和时长不包含“机器人从真实初始位置到作业区”的启动路程。

## 10. 覆盖掩膜与实验指标

覆盖指标不再将矩形包络当作圆盘覆盖，也不依赖低采样率视频位姿。每个连续动作记录车中心起终点、航向、摆臂角、持续时间、动作阶段和 `coverage_active`。对直线行驶同时摆臂的动作，pivot 与圆盘中心分别为

$$
P(t)=p_c(t)-d_pu(\theta(t)),
$$

$$
C(t)=P(t)+L u(\theta(t)+\pi+\phi(t)).
$$

该动作的物理扫掠集合定义为圆盘中心参数曲线与半径 $r_d$ 圆盘的 Minkowski 和：

$$
\mathcal S=\bigcup_{t\in[0,T]}B(C(t),r_d).
$$

实现以固定几何误差递归展平 $C(t)$，再将每段中心折线栅格化为胶囊体。误差阈值由地图像素尺寸和圆盘半径决定，与 pose 的 `sample_rate` 无关。只合并 `coverage_active=true` 动作得到作业圆盘掩膜，合并所有动作得到全轨迹圆盘掩膜。覆盖统计只计入原图中严格等于 255 的自由像素：

$$
N_F=|\{p:M(p)=255\}|,
$$

$$
N_C=|\{p:M(p)=255\land \operatorname{covered}(p)\}|,
$$

$$
N_U=N_F-N_C,\qquad \rho=N_C/N_F.
$$

报告同时给出矩形分割覆盖率、作业圆盘覆盖率和全轨迹圆盘覆盖率。最终 PNG 使用作业圆盘掩膜，因此蓝色自由像素数与 `coverage.work_disc.covered_free_px` 一致。报告中的 `coverage.source` 记录为 `parametric_swept_disc`。

视频仍逐帧累积采样位姿上的圆盘掩膜，并额外绘制黄色车体中心轨迹和红色机器人。因此视频中的蓝色累计区域是低采样率的可视化近似，可能低于 JSON 和最终 PNG 的连续条带覆盖率。

如果调用 `compute_coverage_metrics` 时没有提供连续运动段，它仍会回退到旧的 `sampled_disc_poses` 掩膜。该回退只用于兼容孤立可视化测试，不作为正式实验指标。

## 11. 复杂度分析

令粗栅格尺寸为 $H\times W$，候选矩形数为 $K$，最终矩形数为 $n$，可达节点数为 $n_r$，固定随机路线试验数为 $A=449$。

| 阶段 | 当前实现的主要复杂度 | 说明 |
|---|---:|---|
| 栅格化 | $O(H_pW_p)$ | 每个原像素至多进入一个粗块检查 |
| 行/列候选 | 最坏 $O(HW(H+W))$ | 每个位置可能沿一个轴扫描，再沿另一轴扩展 |
| 贪婪矩形选择 | 最坏约 $O(K^2A_{\max})$ | 每轮遍历剩余候选，并在候选像素切片上求和 |
| 随机补洞 | 与成功矩形数和掩膜重建相关 | 成功次数有限，但没有紧的实用上界；连续失败 5 次停止 |
| 邻接矩阵 | $O(n^2)$ | 两两矩形检查 |
| 连通修复 | 每轮约 $O(n^2+Kn)$ | 可重复至多个桥被加入 |
| Prim MST | $O(n_r^2)$ | 使用邻接矩阵和缓存的最便宜连接 |
| 一次随机贪婪路线 | $O(n_r^3)$ | 每步扫描候选，且对候选重新计算剩余度数 |
| 全局路线搜索 | $O(A n_r^3)$ | $A=449$ 为固定常数 |
| 局部轨迹 | $O(N)$ | $N$ 为输出位姿数 |
| 覆盖掩膜 | $O(Q+A_s)$ | $Q$ 为按固定几何误差展平后的圆盘中心线段数，$A_s$ 为胶囊栅格化面积 |

旧版位掩码动态规划仍以注释形式保留，其状态为“当前节点 + 已访问集合”，空间约 $O(n2^n)$，转移时间可达 $O(n^2 2^n)$。当前 MST + 随机贪婪方法以多项式复杂度换取最优性损失。

|---|---|---:|

## 13. 当前实现能够保证什么、不能保证什么

### 13.1 能够合理声称的性质

- 在固定代码、输入地图、配置和随机种子下，矩形补洞和路线随机试验是确定性的；
- 若起始矩形所在邻接分量非空，MST 分支一定给出覆盖该连通分量全部节点的访问游走；
- 每个矩形第一次访问时都会调用一次矩形内往复式轨迹生成；
- 重复矩形只承担转场，不重复计算内部覆盖；
- 指标图、最终覆盖图和视频的覆盖颜色来自相同的圆盘覆盖逻辑；
- 运行结果记录输入、配置、随机种子、运行时间和产物哈希。

### 13.2 当前不能声称的性质

- 不能声称矩形分解等价于经典 Boustrophedon Cellular Decomposition；
- 不能声称矩形并集完整覆盖全部自由空间；
- 不能声称连通修复总能得到单一连通图；
- 不能声称矩形访问顺序全局最优；
- 不能把 MST 路线直接宣称为标准 TSP 的 2-近似，因为图可能非完全、权重与真实转场不一致，且问题允许节点重访、终点自由；
- 不能声称车体、摆臂和圆盘轨迹无碰撞；
- 不能声称车体和圆盘连续运动过程完全无漏扫；正式指标会如实报告参数化圆盘扫掠后的未覆盖区域，视频仍只是低频采样显示；
- 不能声称满足非完整车式运动学；当前模型允许原地旋转；
- 不能声称最终轨迹时长最小，矩形排序使用基于配置参数的代理代价，且不含真实初始位置到首个入口的路程。

## 14. 论文提交前应重点处理的实现问题

以下不是文字润色问题，而是会影响论文结论可信度的代码事实。

1. **代价模型与实验机器人速度不一致（已修复）。** `computeCoverageCost` 和 `computeTransitionCost` 现在接收当前实验的机器人配置，速度、转向角速度与作业带几何不再使用模块级硬编码常量。

2. **没有碰撞检测（基于研究定位暂不实现）。** 本研究当前侧重阐述分区、排序和局部覆盖的算法思想，不将完整工程级轨迹执行作为贡献范围，因此暂不增加车体/摆臂/圆盘的连续扫掠碰撞检查。论文必须将其作为适用范围限制保留：不声称轨迹无碰撞，并将实验定位为作业端覆盖几何与分层规划方法的验证。

3. **分辨率边界填充与注释不一致（已验证，待修复）。** `resterizeMap` 在布尔障碍掩膜中以 `False` 初始化扩展区，因而将其当成自由空间。对 $3\times5$ 的全自由像素图使用 $g=2$ 时，得到的 $2\times3$ 粗网格全部为自由单元，随机扩展可生成 $(0,0)$–$(6,4)$ 的矩形，超出真实地图边界 $(5,3)$。默认实验使用 $g=1$，因此当前默认结果不受影响。仅将扩展掩膜改为障碍初始化还不足以建立边界不变量，因为后续坐标换算仍使用补齐后的粗网格总尺寸。更稳妥的修复是只构造完全落在原图内的 $\lfloor H/g\rfloor\times\lfloor W/g\rfloor$ 个粗网格，并在候选生成后断言 $0\le x_1<x_2\le W s_p$、$0\le y_1<y_2\le H s_p$。同时应增加非整除尺寸的回归测试。这一保守处理会舍弃右侧或上侧不足一个粗网格宽度的边界带，但可保证矩形不越出已知地图。

4. **候选边界存在末行/末列计数风险。** 行、列候选的 `while` 条件在到达最后索引时不再累计该边界单元，应通过小地图单元测试确认是否存在系统性少算或坐标越界。

5. **`candidate_top_k` 未生效（已修复）。** 该参数现定义为“每条扫描线去重后按面积保留的候选上限”，且必须为正数。它只限制进入贪婪分解的候选；连通修复仍使用完整去重候选池。当前默认值为 20；该取值仍应在更多地图上做敏感性实验。

6. **邻接判据需要统一。** 正常相交分支使用交叠对角线；严格分离分支使用端点距离组合，几何意义不标准。建议改成明确的共享边长度、允许间隙阈值和入口可行宽度判据。

7. **水平边界入口的车中心重建不一致（已修复）。** 入口到起始姿态的过渡现在直接使用 `findRobotPoseByInPoint` 写入的当前车中心，不再在 `fromInPointToStartPoint` 中按航向重新推断入口车中心。

8. **条带间距已显式包含圆盘半径。** 当前条带宽度由 $2(L\sin\psi+r_d)$ 估计，窄条带会自动缩小摆臂摆幅；仍需在论文实验中报告不同 $L,r_d,\psi$ 下的漏扫与越界敏感性。

9. **视频仍是离散可视化。** 正式作业与全轨迹指标已经使用采样率无关的参数化圆盘扫掠；视频仍逐帧画采样位姿，因此视频蓝色累计区域可能低于 `metrics.json` 和 `coverage_final.png`。最终 PNG 与正式作业指标一致。

10. **时长统计是近似值。** 报告用 $(N-1)/f$ 计算总时长，但多个动作段只保存各段终点后的样本，段间采样语义并不完全等价于一条全局均匀时间轴。最好为每个位姿保存显式时间戳并累加动作解析时长。

11. **初始机器人状态被首入口覆盖。** 若论文研究端到端任务时间，应规划配置初始位姿到首矩形入口；否则应明确评价从“首个作业入口已经就位”开始。

12. **地图灰度语义不对称。** 规划时只有值 0 被当作障碍，统计时只有值 255 被当作自由；中间灰度会被规划器视为可行，却不进入覆盖率分母。应在输入阶段二值化并验证。

13. **局部扫描方向与全局覆盖代价不一致。** 图模块假设矩形沿长边作业：宽矩形水平扫、高矩形竖直扫；但标准局部轨迹调用链把主航向设为 $0^\circ/180^\circ$，当前总是水平扫。应显式比较水平与竖直两种局部轨迹代价并选择较优方向，同时让图代价使用同一选择。

14. **覆盖指标已从低采样视频掩膜解耦。** JSON 和最终 PNG 使用固定几何容差下的参数化圆盘扫掠，不再将矩形包络当作覆盖，也不依赖 pose 采样率；视频仍保留逐帧离散圆盘显示，仅作为可视化近似。

15. **摆臂下限已用于固定角速度往复摆动。** 配置默认使用 $[-90^\circ,90^\circ]$，局部覆盖根据条带宽度选择对称摆幅，并以 `arm_angular_velocity_limit` 的固定幅值反复摆动；车辆在必要时降速以让相邻扫掠带重叠。仍需进一步支持非对称摆臂能力的最优利用。

## 15. 建议的论文实验设计

为了让论文不止是实现说明，建议围绕下列研究问题做消融和对比：

### RQ1：双方向矩形候选是否改善分解质量？

比较“仅行扫描”“行 + 列扫描”“行 + 列 + 随机补洞”，报告覆盖率、矩形数、重复面积、分解运行时间。

### RQ2：连通修复是否降低不可达矩形比例？

比较修复前后的连通分量数、起点可达矩形比例、桥接矩形数和新增重叠面积。

### RQ3：双候选路线是否优于单一策略？

比较纯最近邻、随机化贪婪、MST 开放游走、二者择优，报告代理路线代价、真实位姿时长、转弯数和重访数。小规模实例可用注释中的位掩码动态规划恢复精确最优值，计算最优性差距。

### RQ4：机器人几何是否被正确反映到覆盖效果中？

改变 $L,r_d,h_c,d_p$ 和条带间距，报告连续/离散覆盖率、漏扫面积和越界面积。

### RQ5：算法对分辨率和随机性的敏感性如何？

改变 $s_p,g,\ell_{\min},\alpha,\beta$；对补洞阶段使用多个随机种子，报告均值、标准差和最坏值。当前路线随机搜索内部固定种子，如需评估随机性，应将其也暴露为配置参数。

推荐至少加入以下基线：规则栅格 STC、标准 Boustrophedon 分解、仅矩形贪婪不修复、仅 MST 路线，以及小规模精确图覆盖游走。主指标应包括覆盖率、重复覆盖率、路径长度、总时间、转弯次数、重访数、运行时间和碰撞/越界次数。
